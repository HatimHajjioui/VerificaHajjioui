1) Le commit create in una repository locale non vengono sincronizzate automaticamente con la repository online collegata.

2) Il nuovo nome standard del branch principale utilizzato come convenzione in GIT è Main.

3)Il workflow dei comandi di GIT è: git add -> git commit -> git push.

4)Il comando da utilizzare per vedere l'elenco di tutte le commit del branch in cui si trova e i loro hashcode è: git log.

5)Il comando per effettuare il rollback ad una commit precedente è: git reset --hard [hash commit].

6)Git è utilizzato come software di sistema per il controllo delle versioni, consentendo la possibilità di tornare alle versioni precedenti del programma. Rende anche più facile la condivisione di un progetto con tante persone

7)La funzione di git clone è quella di copiare, da una repository online a una repository locale, tutto quello che si e fatto nel branch(in cui viene salvato tutto per esempio le commit). Mentre git pull si usa per esportare un solo file.

8)I rami di sviluppo in GIT sono istanze in cui vengono fatte delle modifiche al progetto che non vanno a modificare il ramo principale(main); si creano con il comando git checkout -b [nome del branch], per spostarsi in un altro ramo si usa il comando git checkout [nome del branch in cui spostarci], per unire le modifiche di un ramo in un altro si utilizza il comando git merge [nome del branch da unire], e per vedere il ramo corrente basta usare git status.

9)L'ingegneria del software si occupa di organizzare in modo professionale un progetto che viene rischiesto da un cliente e deve stare atteneto ai costi,  ai tempi, e alla richiesta del cliente, tutto questo interagisce nella qualità del progetto. 

10) Il ciclo di vita del software è la tutta la storia che compie un software diviso in 5 fasi: l'avvio(start up)(si decide se avviare e accettare un certo progetto), la pianificazione(si organizza e pianifica come verranno svolti i lavori all'interno del gruppo di lavoro), l'esecuzione(la prova del progetto con tutti i test possibili), la manutenzione(quando si deve modificare il software o eventualmente solo fare un controllo così che il software sia sempre utilizzabile), e la chiusura(una volta che il software non viene più utilizzato si elimina).

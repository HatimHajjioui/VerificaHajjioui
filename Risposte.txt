1) Le commit create in una repository locale non vengono sincronizzate automaticamente con la repository online collegata.

2) Il nuovo nome standard del branch principale utilizzato come convenzione in GIT è Main.

3)Il workflow dei comandi di GIT è: git add -> git commit -> git push.

4)Il comando da utilizzare per vedere l'elenco di tutte le commit del branch in cui si trova e i loro hashcode è: git log.

5)Il comando per effettuare il rollback ad una commit precedente è: git reset --hard [hash commit].

6)Git è utilizzato come software di sistema per il controllo delle versioni, consentendo la possibilità di tornare alle versioni precedenti del programma. Rende anche più facile la condivisione di un progetto con tante persone
